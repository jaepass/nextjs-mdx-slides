# Let's touch on the specific features and integrations

---

# Hybrid application

- In-built methods to choose between client and server rendering
- Static pages pre-rendered at build time
- Performance boost
- Server-side rendering for dynamic pages

---

# SSG with ISR

- Hybrid solution to SSG and SSR
- ISR relies on stale-while-revalidate
- Regeneration of static pages during runtime

---

# Page Components & API Routes

- Built-in routing using page components
- Pages are pre-rendered by default
- Build any API endpoint at `pages/api/[endpoint].js`
- Server-side bundles for API endpoints

---

# Serverless Functions and Logging Output

- Handle any backend logic in a serverless function
- Seamless deployment of function to Vercel
- Real-time logging output
- Local development and testing of function

---

# Workflow Integrations

- Headless CMS Strapi integration
- PostGreSQL database integration

---

# Git Integration

- Continuous integration and deployment
- Automatic deploys on every branch push and merge
- Preview deployments

---

# Project features

- Static site generation and Incremental Static Regeneration with Next.js
- Next.js serverless function for form submission
- [Strapi CMS backend](https://github.com/jaepass/strapi-climate-api)
- Strapi Component feature includes:
  - Hero
  - Navigation
  - Content
  - Footer

# ...and more

- Strapi form submission using the create collection type endpoint
- Utility-first CSS with [Tailwind CSS](https://tailwindcss.com)
- Modular page content for all pages, including hero and content blocks
- Page-level SEO settings
